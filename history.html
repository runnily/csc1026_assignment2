Breif History Of The Web

Vannevar Bush (1890-1974) an American Engineer, who introduced the idea of Memex in an article called “As We May Think”. Memex is a theoretical proto-hypertext system. Within the Article, Bush raised his concerns of the increased of evidence being limited “today as specialization extends”. He puts forwards a solution using the theoretical idea of “Memex”. In which he describes it as a “A device in which an individual stores all his books, records and communication”. This influenced the development of future hypertext languages. As later on in the 1960s Doug Englebart and Ted Nelson independently propose idea of “hypertext”. Ted Nelson created the term “hypertext” and founded the first hyper text project (Xanadu) in the 1960. Meanwhile this was all being done and unknown to Englebart.As he too came up with a similar idea. in 1962 he published a report “Augmenting human intellect: A conceptual”. This was a conceptual computer display which shows information from space of linked data. Later on in 1967, ARPANET was developed by the US Advanced Research Project Agency. The initial idea was to communicate and share resources between educative institutions. The idea of internet packs was then introduced, which through the 1970s and 1980s gradually evolved into the internet. Within the 1990s we had what we called the World wide web. This development of the World wide web was greatly influenced by Timothy John Berners Lee, in fact Lee is the scientist who invented the World wide web! In 1980 Lee was working at CERN. This is where he describes the concept of a global system, inspired of the concept of hypertext. Towards the late 80’s Lee took hes idea further and published a paper called “Information Management: A proposal”. Within the article he describes the use of hypertext along with the internet to create a global system used for sharing across information. This was what he called the “World wide web”. Lee also created the first web browser and editor. The world first website was launched on 6 August 1991.

Difference between the web and the internet

The world wide web (W3) is a system of internet servers which supports formatted documents. The documents are formatted in a mark-up language Hyper Text Mark-up Language (HTML). This is a language that supports links to other documents (CSS, java script- We will cover these in the next following sections). As well as to links to documents, it supports graphics, audio and videos files. In simple terms the web is an information system which holds web resources. Each web resources can be identified by a Uniform Resource Locators (URL). The internets is wide global network system, sometimes described as a “networks of networks”. The internet uses TCP/IP to transmit data along various media. The first network developed was a peer-to-peer where each computer would be connected to each other, to share resources. This later gradually enhanced into a wider global network, which -as we mention before- is the internet. The internet is a combination of private, public, business and government technologies. The terms internet and world wide web are sometimes used interchangeably. However, there is a difference. The internet is a global communication system, this includes hardware and infrastructure. On the other hand, the web is the service (web resources etc) passed through the internet. Networking infrastructure allows hardware devices (Servers, Desktop PCS, Mobile etc) to physically connect to one another. The protocols of the internet TCP/IP specify how these communications are carried out. The client (and servers) will allow the user and devices to access and use the internet services.

Seperation of Concerns and Benefits

Before moving on to the other sections. We would need to understand how the documents within a web are formatted. This is done using a hypertext Mark-up language (HTML). HTML is the authoring language using for creating documents (webpages) on the World wide web. The HTML is defining the content structure and semantics (layout) of the webpage by using tags and attributes. HTML can contain links to other text files such as a Cascading style sheet (CSS). A cascading style sheet is used to format the layout of webpages. It defines the text styles and other aspects concerning the presentation of the webpages. CSS has a set of properties to define how the content is display it also includes selectors (defined with the HTML) which selects the elements you want to apply those properties on. The HTML links to another document called Java script. This is an object-oriented programming language to create interactivity on the webpage. In other words, it defines they behaviour of the webpage. JavaScript is a client-side language that has a similar syntax to the object-oriented language Java. There are also other common languages which you could use to define the behaviour such as PHP, Perl, Java and ruby these are all server-side scripting languages, however for this course you would only need to learn about JavaScript. The HTML could link to the other documents. The normally links are CSS and HTML documents- these could also be embedded within the HTML. However, it usually good practice to separate the content (HTML), Presentation (CSS) and behaviour (JavaScript) of a webpage. Separating content and appearance into separate files allows you to easily work on the two separately. It’s simple in terms of maintenance and makes it easier to defines styles on the whole HTML document. Separation can be seen to be more useful, when you have a large website. By using separation, it means you can share the styles across multiple pages from a single CSS file.
