Web Forms

Before we start going into the behaviour aspects of websites, we will cover a little bit of CSS, specifically web forms. Learning web forms will be useful alongside java script. Web forms are form tags and attributes used to create an interface for user interaction. The core purpose of a form is to collect the user’s information and then return it to the server. The form specifies how the browser should include use that data. The form structure is specified using the <form> tags. The method and action attributes should be included in the form. Below is an example of how the form tag should look like: <body> <form method = “get” action= http://a.server.com/script> <p> Enter your name: <input type= “text” name= “username”/> </p> </form> <body> Form controls are replaced tags used to collect user input. When rendered the tag is replaced by the browsers interface for the control type. Each form control will have a name attribute, and most will have a value attribute. On a successful submission the form controls name/value pairs would be given to the server. For example checkboxes is a type of form control, the value selected would go to the server. Types of Buttons

There are three types of form control buttons. These submit button makes the browser submit the data to the server. Here is an example of the submit button: <input type=“submit” value=“submit this info” />The reset is a button which clears the users input and sets the values to its initial state without reloading the page. The next form control button is simply called button. This is a generic button, which can be only used when attached to a client-side scripted actions or events.

The Submit Process

When a button is clicked the name/value pairs within the forms are attached to a request within the URL. The URL is specified by the action attribute in the form tag. The method attribute would specify what type of request is going to be made. There are two types of methods. This includes the GET method and POST method. The GET method is the default method for all web request. This appends the data to the end of the URL query in a query string. This means the user input can be seen -which is why the get method is not usually used for sensitive data. The processing script will take a look in the query string and extract the data it needs. This is an example of a query string http://.../formscript.php?inputbox=Arnold&choice=A. The post method specifies that the data should be carried within the body of the request. Therefore, the URL would not contain the query string and would look something like this: http://.../formscript.php. The post method is not a default method, so it needs to be invoked by specifying the method “post”. Once the user has interacted with the form, the results are now processed. There are 3 stages involving the processing. This includes validation, processing and feedback. The validation checks that the forms have been completed and has the correct type of data. Processing is using the data within the form and doing something useful with it. The feedback returns something to user. The validation can be done on a client side (java script) or server side (python, php etc). However, for this module you would only need to understand the client side.

JavaScript Programming recap

Java script is an object-oriented scripting language that is commonly used to create interactive behaviour effects within web browsers. The language is executed in the web browser on the client machine. Hence the name “client side” scripting. Java script could be added to pages using the inline <script> <script type= “text/javascript”> document.write(“<h1>Hello world</>”); </script> or using an external script by specifying the src attribute within the <script> tag <script type= “text/javaScript” src= “scripts/myjava.js”> </script> Your <script> can be placed within the <head> or <body> tag of your html.
