HTML: Indroduction

HTML was made with the idea to simply design a handle text-based document. As the popularity of the web increased, there was a need for new features to be added. This meant improving organisation and presentation. In the late 80’s Berners-Lee proposed the concept of an internet-based hypertext system. Lee specified HTML and wrote the browser in the late 1990. HTML2.0 was published in 1995. Future web enhancement meant HTML3.2 was later introduced and became standardised common hypertext language for structuring web pages. Towards the late 90’s another version of HTML was introduced- HTML4.01. This added more enhancements for the new technologies. This included: support for CSS, Scripting and accessibly features. The problem with this HTML is that it was only targeted at browsers which were on PCs and work stations. This meant the syntax used to write the HTML was quite lenient (errors and variations in the HTML mark-up, elements without tags etc). This meant that browser had much more processing to do, as the mark-up language contained many variations. XHTML 1.0 was published around 2000 to overcome this. This language was a mixture of XML (Extensible Mark-up Language- This is a set of rules that defines encoding of documents). XHTML had the same syntax as HTML, however with the XML it provided a stricter syntax. This meant tags, attributes were lowercase, Attributes must have a name value pairs, Tags must open, and close, self-contained tags must be closed etc. With XHTML meant more device accessibility and browser used less processing power to deal with the syntax. This also pushed forward the idea of enforced separation. HTML 5 was later introduced. This improved semantic organisations, device independence and removed the need for browser plugins

HTML: DTD

The specification for an HTML document is provided by the World Wide Web Consortium (W3C). W3C defines the document type definition (DTD) for each version of HTML. A DTD defines the rules of each version of the HTML (XHTML, HTML4.01, HTML5). It defines: The tags which can be used within the document, where they are supposed to appear in the document, which attributes they can hold. This website provides good examples of the differences between the version of HTML. The basic document html structure is specified by W3C “1. a line containing HTML version information, 2. a declarative header section (delimited by the HEAD element), 3. a body, which contains the document's actual content. The body may be implemented by the BODY element or the FRAMESET element.” The section 2 and 3 are within the html tag. Below is an example <html> <head><title>My first HTML document</title></head> <body><p><Hello World!</p></body> </html> The main purpose of the DTD is to define the document version which the user is using. Below is an example of a doctype for XHTML1.0 <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"> For HTML4.02: <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"> For HTML5 its simply just: <!DOCTYPE html> As mention before the document contains the a head, which is for information about the page and not for display: <head> <title>Title of the document</title> </head> Here you could specify optional declarations using the <meta> tag and <script> </script> to define a script or <style></style> to define the CSS. We’ve recently touched on the importance of concerns of separation so it would be better if we used <link> </link> tag to link script or CSS file. Body Tag is used for content display, An example is shown below: <body> The content of the document...... </body> Overall the using an example of HTML5 the HTML document should appear like this : <!DOCTYPE html> <html> <head> <title>Title of the document</title> </head> <body> <p>Content here</p> </body> </html>

Types of Tags and Attributes

In the other section we came across the <head>, <html>, <body> and <html>. These are known as tags which are wrapped around content in pairs such as this <h1> This is a heading </h1>.The tags can be categorised into types of Tags. The block-level tag. This tag provides basic content structure for the webpage and defines blocks of content. Heading <h1> </h1> and paragraphs <p></p> are examples of block level tags. The browser will treat these tags as the name suggest -a block. These adds new lines above and below. Inline tags are tags which can be nested within a block. These tags do not create a new line <p> The <em> usefull </em> bit of the paragraph. There are some tags which pairs are not <br />. Tags contain attributes within them. Attributes specify additional properies for the HTML tags. For example <img src = “logo.gif” alt = “my logo”> is an example of a tag. The common attributes includes: Class, this attributes associates element with a CSS class Id, this will uniquely identify elements for CSS or scripting Style will specify the style of the tag with the CSS.

Standards and Validation

Web standards are defined as “technologies used for creating and interpreting web-based content [design and develop to] ensure long-term viability of web publishing”. This empathise the fact that you should use web standards, to allow your website to have a long-term viability. Standards are extremely important within the industry because they allow the combination of products from different manufacturers to create a customized system. HTML is derived from Standard Generalized Mark-up Language (SGML). This is a standard for specifying a document mark-up language or tag set. It’s important to understand the SGML is not a language itself, but a formal description of the language. Mark-up languages should contain a DTD declaration. All versions of HTML have at least two DTDs associated with it, these includes, Strict DTD, This contains only current and valid tags and attributes. Traditional DTD retains deprecated definitions for older mark-up tags. To provide backwards combability.
